# 와이어프레임 초안

## 화면 원칙

이 앱의 첫 화면은 소개 페이지가 아니라 바로 사용할 수 있는 경기 운영 화면이어야 한다.

운영자는 경기 중에도 직접 게임에 참여할 수 있으므로 조작은 짧고 커야 한다. 멤버는 카카오톡 오픈채팅방 링크로 들어와 현재 경기와 본인 순서를 빠르게 확인한다.

핵심 UX 원칙:

- 모바일 우선
- 큰 터치 영역
- 한 화면에서 현재 상태 파악
- 운영자 기능과 멤버 조회 기능 분리
- 멤버 화면에는 점수/운영진/면제 정보 노출 금지
- 카카오톡 인앱 브라우저에서도 문제 없는 단순한 상호작용

## 정보 구조

```text
/admin/new
  새 경기 세션 준비

/admin/:sessionId?key=...
  운영자 경기 진행

/board/:sessionId
  멤버용 경기판
```

## 운영자 플로우

```text
1. 회원명단 불러오기
2. 참석자 선택
3. 필요 시 참석자별 참여 빈도 설정
4. 코트 수 설정
5. 첫 코트 배정 생성
6. 공개 링크 공유
7. 코트별 경기 시작 처리
8. 코트별 경기 종료 처리
9. 종료된 코트에 대기 큐 앞쪽 사람들을 자동 배정
10. 필요 시 대기 큐 순서를 드래그로 변경
11. 필요 시 대기자에게 우선권 또는 보류 상태 부여
```

## 멤버 플로우

```text
1. 카카오톡 링크 접속
2. 현재 경기 확인
3. 내 이름 검색 또는 선택
4. 내 상태 확인
   - 지금 경기
   - 다음 경기
   - 대기
5. 자동 갱신으로 다음 라운드 확인
```

## 화면 1. 운영자 새 세션

목적: 오늘 참석자를 빠르게 고르고 첫 라운드를 생성한다.

```text
┌──────────────────────────────┐
│ 꿈꾸민턴        오늘 경기     │
├──────────────────────────────┤
│ 코트 [-] 3 [+]                │
├──────────────────────────────┤
│ 참석 24명   경기 6명   대기 0명│
├──────────────────────────────┤
│ [ 회원 검색                 ] │
├──────────────────────────────┤
│ □ 김도윤   남 A 82 0게임 일반  │
│ □ 박수란   여 B 76 1게임 높음  │
│ □ 이지훈   남 C 61 0게임 낮음  │
│ □ 최유진   여 D 48 2게임 일반  │
│                              │
│ 선택된 사람은 행 전체 강조     │
├──────────────────────────────┤
│ [선택 보기 24] [라운드 생성]   │
└──────────────────────────────┘
```

### 구성 요소

- 상단 상태 영역
- 코트 수 stepper
- 회원 검색
- 회원 리스트
- 선택된 참석자 count
- 하단 고정 action bar

### 회원 리스트 표시

운영자 화면에서는 매칭 판단을 위해 최소 정보를 보여준다.

```text
회원명 / 성별 / 급수(지역) / 점수(백분위) / 오늘 경기 수 / 참여 빈도
```

점수는 공개 경기판에는 표시하지 않는다.

### 참여 빈도 설정

참여 빈도는 오늘 세션 안에서만 쓰는 운영자 설정이다. 기본값은 `일반`이다.

```text
높음: 더 자주 치고 싶은 참석자
일반: 기본값
낮음: 게임 수가 많으면 부담되는 참석자
```

참석자 목록에서 이름을 누르면 상세 시트로 빈도를 바꿀 수 있다.

```text
┌──────────────────────────────┐
│ 박수란                        │
├──────────────────────────────┤
│ 오늘 1게임                    │
│ 참여 빈도                     │
│ [높음] [일반] [낮음]          │
├──────────────────────────────┤
│ [닫기]                        │
└──────────────────────────────┘
```

## 화면 2. 선택 참석자 확인

목적: 라운드 생성 전 빠르게 선택 오류를 제거한다.

```text
┌──────────────────────────────┐
│ 선택 참석자 24명              │
├──────────────────────────────┤
│ 남 14명   여 10명             │
│ 3코트 기준 현재 배정: 3게임     │
├──────────────────────────────┤
│ 김도윤   [삭제]               │
│ 박수란   [삭제]               │
│ 이지훈   [삭제]               │
│ 최유진   [삭제]               │
├──────────────────────────────┤
│ [닫기]              [생성]    │
└──────────────────────────────┘
```

### 표시 상태

- 총 참석자 수
- 남/여 수
- 현재 코트 수 기준 생성 가능한 게임 수
- 삭제 버튼

## 화면 3. 운영자 코트 운영판

목적: 코트별 현재 상태를 확인하고, 운영자가 직접 시작/종료를 누른다.

```text
┌──────────────────────────────┐
│ 오늘 경기       공유 링크 복사 │
├──────────────────────────────┤
│ 운영 중 3코트   대기 12명       │
├──────────────────────────────┤
│ 1코트 · 진행중                 │
│ 김도윤(2) / 박수란(1)          │
│        vs                      │
│ 이지훈(1) / 최유진(2)          │
│                    [종료]      │
│                                │
│ 2코트 · 배정됨                 │
│ ...                            │
│           [재배정] [시작]      │
│                                │
│ 3코트 · 비어있음               │
│ 대기자 4명 이상이면 배정 가능   │
│                    [배정]      │
├──────────────────────────────┤
│ 대기 큐                        │
│ ≡ 1. 강민수 0게임 높음         │
│ ≡ 2. 오하나 1게임 일반         │
│ ≡ 3. 정지우 0게임 낮음         │
│ ≡ 4. 한수민 2게임 일반         │
│ 드래그로 순서 변경 가능         │
├──────────────────────────────┤
│ [대기 편집] [결과 복사] [링크]  │
└──────────────────────────────┘
```

### 주요 액션

- 코트별 시작
- 코트별 종료
- 배정됨 상태 재배정
- 대기 큐 편집
- 대기 큐 드래그 순번 변경
- 대기자 우선권/보류 지정
- 참석자별 오늘 경기 수 확인
- 참석자별 참여 빈도 설정
- 결과 복사
- 공개 링크 복사

### 코트 상태와 배정 정책

대기자는 특정 코트나 미래 경기 조에 미리 배정하지 않는다. 하나의 개인 대기 큐만 유지하고, 어떤 코트든 먼저 완료되면 그 코트에 대기 큐 앞쪽 사람들을 우선해 4명을 구성한다.

```text
1코트 종료 -> 대기 큐 앞쪽 사람들로 1코트 다음 경기 자동 배정
3코트 종료 -> 대기 큐 앞쪽 사람들로 3코트 다음 경기 자동 배정
```

코트 상태:

```text
비어있음: 현재 배정 없음
배정됨: 4명이 코트에 배정됐지만 아직 시작 전
진행중: 운영자가 시작을 누른 실제 진행 경기
```

### 진행 중 경기 변경 정책

진행 중인 코트는 재배정하지 않는다. 운영자가 `시작`을 누른 경기는 실제 진행 중인 게임으로 보고, 운영자 화면에서도 선수 이동/교체/팀 재계산을 제공하지 않는다.

조정이 필요한 경우에는 대기 큐 순서와 상태만 바꾼다. 다음 경기는 어떤 코트든 먼저 완료되는 순간 대기 큐 기준으로 새로 구성된다.

## 화면 4. 운영자 대기 큐 편집

```text
┌──────────────────────────────┐
│ 대기 큐 편집                  │
├──────────────────────────────┤
│ 다음 빈 코트는 이 순서 기준     │
├──────────────────────────────┤
│ ≡ 1. 강민수 0게임 높음 [우선]  │
│ ≡ 2. 오하나 1게임 일반 [보류]  │
│ ≡ 3. 정지우 0게임 낮음 [우선]  │
│ ≡ 4. 한수민 2게임 일반 [보류]  │
├──────────────────────────────┤
│ 보류 중인 사람은 다음 배정 제외 │
│ 우선권은 대기 큐 앞쪽처럼 계산  │
├──────────────────────────────┤
│ [취소]              [저장]    │
└──────────────────────────────┘
```

목적: 운영자가 실제 현장 상황에 맞춰 대기 큐만 조정한다. 이미 진행 중인 코트의 선수는 이 화면에 편집 대상으로 나오지 않는다.

### 대기 큐 편집 정책

- 운영자는 대기 큐에서만 사람을 이동할 수 있다.
- 대기자는 특정 코트에 미리 배정되지 않는다.
- 우선권이 있는 대기자는 다음 빈 코트 배정 계산에서 앞쪽 후보로 본다.
- 보류 상태인 대기자는 잠시 다음 배정 후보에서 제외하거나 큐 뒤쪽으로 보낸다.
- 참여 빈도 `높음/일반/낮음`은 오늘 경기 수와 함께 매칭 우선순위를 보정한다.
- 진행 중 경기의 선수는 대기 큐 편집 대상이 아니다.

```text
가능: 대기 큐 4번 한수민을 대기 큐 1번으로 이동
가능: 대기 큐 2번 오하나를 보류 상태로 변경
불가: 2코트에서 경기 중인 한수민을 1코트로 이동
불가: 대기 중인 한수민을 1코트 예정자로 고정
```

## 화면 5. 대기 큐 순번 변경

목적: 운영자가 실제 현장 상황에 맞춰 대기 순서를 빠르게 조정한다.

```text
┌──────────────────────────────┐
│ 대기 큐 편집                  │
├──────────────────────────────┤
│ ≡ 1. 강민수                   │
│ ≡ 2. 오하나                   │
│ ≡ 3. 정지우                   │
│ ≡ 4. 한수민                   │
│                                │
│ 손가락으로 잡아 위아래 이동     │
├──────────────────────────────┤
│ [취소]              [저장]    │
└──────────────────────────────┘
```

대기 큐 순번 변경은 운영자 의도를 강하게 반영한다. 이후 빈 코트 배정은 변경된 대기 큐 순서를 우선한다.

## 화면 6. 멤버용 경기판

목적: 멤버가 현재 경기와 본인 순서를 빠르게 확인한다.

```text
┌──────────────────────────────┐
│ 꿈꾸민턴 경기판               │
│ 방금 업데이트됨               │
├──────────────────────────────┤
│ [내 이름 검색               ] │
├──────────────────────────────┤
│ 내 상태                       │
│ 박수란 · 1코트 · 진행중        │
│ 파트너 김도윤                 │
│ 상대 이지훈 / 최유진           │
├──────────────────────────────┤
│ 코트 현황                     │
│ 1코트 김도윤/박수란 vs ...     │
│ 2코트 배정됨 · 시작 전         │
│ 3코트 비어있음                 │
├──────────────────────────────┤
│ 대기 큐                        │
│ 1 강민수  2 오하나  3 정지우   │
└──────────────────────────────┘
```

### 멤버 화면에서 숨길 정보

- 점수(백분위)
- 급수(지역)
- 오늘 경기 수
- 참여 빈도
- 가입일
- 운영진 여부
- 면제 여부
- 비고

## 화면 7. 내 이름 검색 결과

목적: 멤버가 “내가 언제 치는지”만 바로 알 수 있게 한다.

```text
┌──────────────────────────────┐
│ [박수란                    ] │
├──────────────────────────────┤
│ 검색 결과                     │
│ ○ 박수란                      │
│ ○ 박서란                      │
├──────────────────────────────┤
│ 선택 후 localStorage 저장      │
└──────────────────────────────┘
```

선택한 이름은 브라우저 localStorage에 저장한다. 다음 접속 때 자동으로 해당 멤버 상태를 강조한다.

## 상태 표현

### 지금 경기

```text
진행중
1코트
파트너: 김도윤
상대: 이지훈 / 최유진
```

### 배정됨

```text
시작 전
2코트에 배정되었습니다.
운영자가 시작을 누르면 진행중으로 바뀝니다.
```

### 다음 경기

대기자는 특정 코트나 미래 경기 조에 미리 배정되지 않으므로 “2코트 예정”처럼 표시하지 않는다. 대신 대기 큐 순서와 다음 빈 코트에 들어갈 가능성을 보여준다.

```text
대기 3번
앞에 2명이 대기 중입니다.
먼저 비는 코트에 순서대로 배정됩니다.
```

### 대기

```text
대기 중
현재 코트에는 배정되지 않았습니다.
```

## 빈 상태

### 회원명단 로딩 실패

```text
회원명단을 불러오지 못했습니다.
잠시 후 다시 시도해주세요.
```

### 참석자 4명 미만

```text
경기를 만들려면 최소 4명이 필요합니다.
```

### 대기 큐 없음

```text
대기자가 없습니다.
모든 선택 참석자가 현재 경기에 배정되었습니다.
```
